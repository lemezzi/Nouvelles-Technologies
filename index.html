Bonnes pratiques

Étapes clés pour écrire un Code Propre

1. Nommer clairement

Utiliser des noms explicites pour les variables, fonctions et classes. Le nom doit dire ce que fait l’élément.

2. Garder des fonctions courtes

Une fonction doit faire une seule chose et bien la faire. 10-15 lignes max si possible.

3. Structurer son code

Organiser les fichiers et dossiers de manière cohérente. Séparer la logique métier, les composants, les tests...

4. Éviter le code mort

Supprimer tout code inutile, non utilisé ou commenté sans but. Il pollue la lisibilité.

5. Tester pour mieux coder

Les tests (unitaires, d'intégration) renforcent la qualité et facilitent les refactorings.

Développeur Débutant

Code Confus et Inefficace 
    int i,j,t;
    int x[5] = {5,2,9,1,3};
    for(i=0;i<5;i++)
        for(j=0;j<5;j++)
            if(x[i]<x[j]){
                t=x[i];
                x[i]=x[j];
                x[j]=t;
            }
    for(i=0;i<5;i++) printf("%d ",x[i]);
  • ❌ Mauvaise indentation et lisibilité
  • ❌ Aucune fonction ni modularité
  • ❌ Complexité élevée et inefficace
  • ❌ Difficile à maintenir et à comprendre

Développeur Moyen

Fonctionnel, mais Améliorable 
    void bubbleSort(int arr[], int n){
        for(int i=0; i<n-1; i++)
            for(int j=0; j<n-i-1; j++)
                if(arr[j] > arr[j+1]){
                    int temp = arr[j];
                    arr[j] = arr[j+1];
                    arr[j+1] = temp;
                }
    }
    int data[5] = {5,2,9,1,3};
    bubbleSort(data, 5);
    for(int i=0; i<5; i++) printf("%d ", data[i]);
  • ✔️ Utilisation d'une fonction dédiée
  • ⚠️ Manque de documentation et de modularité
  • ⚠️ Algorithme inefficace (O(n²))
  • ⚠️ Peu évolutif pour des grands ensembles

Développeur Expérimenté

Lisible, Rapide et Evolutif 
    /**
     * Trie un tableau en utilisant le tri rapide
     * @param arr - tableau à trier
     * @param low - indice de début
     * @param high - indice de fin
     */
    void quickSort(int arr[], int low, int high) {
        if (low < high) {
            int pivot = partition(arr, low, high);
            quickSort(arr, low, pivot - 1);
            quickSort(arr, pivot + 1, high);
        }
    }
    // Exemple d'utilisation
    int arr[5] = {5,2,9,1,3};
    quickSort(arr, 0, 4);
    for(int i=0; i<5; i++) printf("%d ", arr[i]);
  • ✅ Algorithme efficace (O(n log n))
  • ✅ Documentation claire et explicite
  • ✅ Fonction modulaire et réutilisable
  • ✅ Performance et lisibilité maximisées
